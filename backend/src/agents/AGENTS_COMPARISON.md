# Player Analysis Agents 功能对比

## 📊 11个Agents功能总览

### 原始5个 - 时间维度完整覆盖
| Agent | 数据范围 | 分析角度 | 输出长度 | 核心价值 |
|-------|---------|---------|---------|---------|
| **PostgameReviewAgent** | 单场比赛 | 单局诊断 | 规则引擎 | 赛后复盘量化诊断 |
| **DetailedAnalysisAgent** | 1个版本 | 微观深度 | 8000-10000字 | 逐版本/逐英雄详细剖析 |
| **VersionComparisonAgent** | 2个版本 | 横向对比 | 1000-1500字 | Coach Card + 版本间变化分析 |
| **MultiVersionAgent** | 3-9个版本 | 宏观趋势 | 1000字 | 跨版本适应能力评估 |
| **AnnualSummaryAgent** | 40-50个版本 | 年度总结 | 3000-5000字 | 整个赛季综合分析 |

### 新增6个 - 维度扩展
| Agent | 数据维度 | 分析角度 | 输出长度 | 核心价值 |
|-------|---------|---------|---------|---------|
| **ChampionMasteryAgent** | 单英雄全部历史 | 英雄掌握度 | 2000-3000字 | 学习曲线/位置专精/掌握度评分 |
| **RoleSpecializationAgent** | 单位置全部历史 | 位置专精 | 2500-3500字 | 英雄池广度/深度/位置评分 |
| **ProgressTrackerAgent** | 最近10-20版本 | 进步追踪 | 2000-2500字 | 前半/后半对比/学习速度 |
| **WeaknessAnalysisAgent** | 最近5-10版本 | 弱点诊断 | 1500-2000字 | 对线/团战/宏观弱点识别 |
| **PeerComparisonAgent** | 玩家vs段位基准 | 同段位对比 | 2000-2500字 | 相对定位/优劣势识别 |
| **ChampionRecommendationAgent** | 英雄池+Meta | 英雄推荐 | 1500-2000字 | 风格匹配/Top5推荐 |

## 1️⃣ MultiVersionAgent - 多版本趋势分析

**定位**: 宏观战略视角的跨版本适应分析

### 输入数据
- 9个版本的 player-pack 数据
- 时间跨度：15.12 → 15.20

### 核心功能
1. **趋势识别**：识别核心英雄池的胜率趋势
2. **转折点分析**：游戏量激增、英雄池变化的关键版本
3. **适应能力评估**：玩家在版本变化中的适应性评级
4. **英雄池规划**：推荐保留/放弃的英雄

### 输出示例
```
# 跨版本适应能力分析报告

## 核心英雄池分析
- **稳定英雄**: 英雄92（TOP/MIDDLE/BOTTOM多位置）
- **上升英雄**: 英雄92 (BOTTOM)，胜率从50%→66.7%
- **下滑英雄**: 英雄76 (TOP)，胜率从75%→0%

## 关键转折点
1. **15.16 → 15.17**: 游戏量激增522%
```

### 独特价值
- ✅ 识别长期趋势（不关注单个版本细节）
- ✅ 发现版本适应模式（快速响应 vs 滞后调整）
- ✅ 轻量快速（Haiku模型，4000 tokens）

## 2️⃣ DetailedAnalysisAgent - 详细深度分析

**定位**: 微观战术视角的全面深度剖析

### 输入数据
- 多个版本的 player-pack 数据
- 全局 meta 文件（delta_cp, item/rune changes）

### 核心功能
1. **逐版本详细分析**：每个版本的英雄选择策略、胜率表现
2. **核心英雄全面剖析**：每个英雄的跨版本表现轨迹、出装变化
3. **出装与符文深度解析**：哪些装备调整成功/失败
4. **Meta适应性评估**：玩家选择 vs 全局Meta的对齐程度
5. **数据驱动战术建议**：具体的装备搭配、训练重点

### 输出示例
```
# 详细深度分析报告 (8000-10000字)

## 二、逐版本深度分析
### 15.18版本
- 游戏量: 52场
- 英雄池策略: 主打亚索(MIDDLE)和剑姬(TOP)
- 胜率表现: 53.8%
- 与15.17对比: 游戏量下降15%，胜率提升2.1%
- 亮点: 亚索胜率从45%→62%
- 问题: 剑姬连续4场失败

## 三、核心英雄全面剖析
### 英雄92 (亚索)
- 跨版本表现: 15.12(50%) → 15.17(78.3%) → 15.20(66.7%)
- 胜率波动原因: 15.17版本Q技能CD降低，出装从破败→无尽优先
- 出装变化:
  - 15.16前: 破败→攻速鞋→无尽
  - 15.17后: 无尽→狂战士胫甲→破败
- 优势版本: 15.17 (Meta buff + 个人表现提升)
- 调整建议: 保持无尽优先出装，15.20版本考虑回归破败流
```

### 独特价值
- ✅ 每个版本的详细诊断（而非趋势概览）
- ✅ 每个英雄的深度剖析（出装进化、符文选择）
- ✅ Meta对齐分析（英雄buff但个人表现下滑 = 打法问题）
- ✅ 超详细报告（8000-10000字，Sonnet 16000 tokens）

## 3️⃣ VersionComparisonAgent - 双版本对比 (Coach Card)

**定位**: 版本间横向对比 + 教练指导卡

### 输入数据
- 两个版本的 player-pack（15.18 vs 15.19）
- 全局 meta 变化（delta_cp, item changes, rune changes）

### 核心功能
1. **全局Meta变化总结**：Top 10 buffed/nerfed 英雄、装备、符文
2. **玩家适配分析**：英雄池变化、出装适配、符文适配
3. **关键建议生成**：
   - 双重劣势英雄（表现下滑 + 英雄nerf）→ 建议更换
   - 英雄buff但表现下滑 → 建议调整打法/build
   - 表现提升 + 英雄buff → 继续使用
4. **完整Coach Card**：JSON格式的版本对比数据包

### 输出示例
```
# 版本对比报告 (1000-1500字)

## 版本变化总结 (15.18 → 15.19)
- Buffed英雄: 剑姬(+0.45 CP), 亚索(+0.32 CP)
- Nerfed英雄: 刀妹(-0.51 CP), 锐雯(-0.38 CP)
- 装备变化: 破败王者之刃(+2.3 GE), 无尽之刃(-1.1 GE)

## 玩家表现分析
### 英雄池变化
- 亚索 (MIDDLE): 胜率 62% → 58% (下降4%)
  - Meta: 英雄buff (+0.32 CP)
  - 诊断: **英雄被buff但表现下滑** → 打法/出装问题
  - 建议: 检查出装路线，考虑从无尽流回归破败流

### 装备与符文建议
1. **破败王者之刃** (+2.3 GE): 优先级提升
   - 适用英雄: 亚索、剑姬、刀妹
2. **无尽之刃** (-1.1 GE): 效率下降
   - 建议: 延后出装顺序
```

### 独特价值
- ✅ 版本间横向对比（而非纵向趋势）
- ✅ Meta变化量化（delta_cp, delta_ge）
- ✅ 完整Coach Card JSON（可供其他系统使用）
- ✅ 教练指导建议（swap/retune/keep）

## 4️⃣ PostgameReviewAgent - 赛后复盘

**定位**: 单场比赛的量化诊断系统

### 输入数据
- 单场比赛的 match_features（match_id, champion, role, win, kda等）
- 单场比赛的 timeline_features（cs_at, gold_curve, item_purchases等）

### 核心功能
1. **对线期诊断**：CS@10, Gold@10, 回家效率
2. **目标控制诊断**：视野布置、目标参与率
3. **出装节奏诊断**：二件套时间、装备数量
4. **团战表现诊断**：KDA、参团率、死亡次数
5. **A-D评分系统**：问题越少分数越高

### 输出示例
```json
{
  "match_id": "NA1_1234567890",
  "champion": "亚索",
  "role": "MIDDLE",
  "result": "WIN",
  "lane_phase": {
    "cs10": 65,
    "gold10": 3200,
    "issues": []
  },
  "objective_phase": {
    "wards_placed": 3,
    "obj_participation": 3,
    "issues": []
  },
  "build_timing": {
    "core2_time": 19.2,
    "items_count": 3,
    "issues": []
  },

    "kda": 4.0,

    "score": 100,
```

### 独特价值
- ✅ 单场比赛诊断（而非多场统计）
- ✅ 基于规则的量化诊断（无需LLM即可运行）
- ✅ 可选LLM增强叙述（Sonnet/Haiku）
- ✅ 即时反馈（赛后立即诊断）

## 5️⃣ AnnualSummaryAgent - 年度赛季总结

**定位**: 整个赛季的综合分析与年度总结

### 输入数据
- 40-50个版本的 player-pack 数据
- 时间跨度：整个赛季（约10-12个月）

### 核心功能
1. **时间分段分析**：月度/季度/三期表现对比
2. **年度亮点提取**：最佳表现时期、最佳英雄-位置组合、里程碑成就
3. **版本适应趋势**：关键版本转折点、适应模式评估
4. **英雄池演进**：核心英雄池稳定性、新英雄学习曲线、英雄池广度变化
5. **成长轨迹描述**：早期/中期/晚期三段对比
6. **未来展望与建议**：基于赛季表现的优势分析和改进建议

### 输出示例
```markdown
# 2025赛季年度总结报告

## 一、赛季总览
```

## 🎯 功能重叠度分析

### MultiVersionAgent vs DetailedAnalysisAgent
**重叠度**: 低（10%）

- **相同点**：都是多版本分析
- **不同点**：
  - MultiVersionAgent：宏观趋势（哪些英雄在上升/下降）
  - DetailedAnalysisAgent：微观深度（为什么上升/下降，具体数据）

**使用场景**：
- 想快速了解跨版本适应能力 → MultiVersionAgent
- 想深入分析每个版本的具体情况 → DetailedAnalysisAgent

---

### MultiVersionAgent vs VersionComparisonAgent
**重叠度**: 极低（5%）

- **相同点**：都涉及多版本数据
- **不同点**：
  - MultiVersionAgent：9个版本的时间序列趋势
  - VersionComparisonAgent：2个版本的横向对比 + Meta变化

**使用场景**：
- 想看长期趋势和适应模式 → MultiVersionAgent
- 想对比两个版本的具体变化 → VersionComparisonAgent

---

### DetailedAnalysisAgent vs VersionComparisonAgent
**重叠度**: 中等（30%）

- **相同点**：都包含Meta对齐分析、出装变化分析
- **不同点**：
  - DetailedAnalysisAgent：纵向深度挖掘（每个英雄的详细轨迹）
  - VersionComparisonAgent：横向版本对比（Meta变化、Coach Card）

**使用场景**：
- 想深入了解每个英雄的表现细节 → DetailedAnalysisAgent
- 想快速对比两个版本的变化 → VersionComparisonAgent

## 📋 总结与建议

### 5个Agents是否重复？

**答案：不重复，各有独特价值**

1. **MultiVersionAgent** (多版本趋势)
   - 独特价值：长期趋势、适应模式、版本转折点
   - 不可替代：轻量快速、宏观视角、战术级分析（1-2个月）

2. **DetailedAnalysisAgent** (详细深度分析)
   - 独特价值：逐版本深度剖析、每个英雄的详细轨迹
   - 不可替代：超详细报告（8000-10000字）、Meta对齐深度分析

3. **VersionComparisonAgent** (双版本对比)
   - 独特价值：版本间横向对比、Coach Card、Meta变化量化
   - 不可替代：教练指导建议（swap/retune/keep）

4. **PostgameReviewAgent** (赛后复盘)
   - 独特价值：单场比赛诊断、即时反馈、规则引擎
   - 不可替代：赛后即时复盘、4维度量化诊断

5. **AnnualSummaryAgent** (年度赛季总结)
   - 独特价值：整个赛季的综合分析、时间演进叙事、成长轨迹
   - 不可替代：战略级年度总结（40-50个版本）、3000-5000字叙事报告

### 推荐使用组合

**场景1: 完整玩家分析流程**
1. AnnualSummaryAgent → 整个赛季的年度总结
2. MultiVersionAgent → 了解近期长期趋势
3. DetailedAnalysisAgent → 深入分析具体问题
4. VersionComparisonAgent → 对比关键版本变化
5. PostgameReviewAgent → 单场复盘改进

**场景2: 快速版本适配**
1. VersionComparisonAgent → 了解版本变化
2. PostgameReviewAgent → 新版本单场复盘

**场景3: 深度玩家报告**
1. MultiVersionAgent → 宏观适应评估
2. DetailedAnalysisAgent → 详细数据支撑

**场景4: 年度总结报告**
1. AnnualSummaryAgent → 整个赛季综合分析
2. MultiVersionAgent → 关键时期的深入分析

## 6️⃣ ChampionMasteryAgent - 英雄掌握度分析 ✨ NEW

**定位**: 单英雄的深度掌握度评估

### 输入数据
- 指定英雄ID的全部历史数据（跨所有版本）
- 例如：英雄92在9个版本中的175场比赛数据

### 核心功能
1. **学习曲线分析**：早期(1-30场) / 中期(31-100场) / 后期(101+场)三阶段表现对比
2. **位置专精度**：该英雄在不同位置的表现（TOP/MID/ADC等）
3. **版本适应性**：跨版本的稳定性和表现趋势
4. **掌握度评分**：S/A/B/C/D/F评级系统（基于游戏量+胜率+稳定性+版本覆盖）
5. **优势/改进建议**：基于数据的具体建议

### 输出示例
```markdown
# 英雄掌握度报告 - 英雄ID 92

## 掌握度评分: A-

**总场次**: 175场（占总游戏52.7%）
**整体胜率**: 58.3% (95% CI: 51.2% - 65.1%)
**版本覆盖**: 9/9个版本（100%）
**主要位置**: 中路（67.6%胜率，37场）

## 学习曲线分析
- **早期（1-30场）**: 52.1%胜率，探索期
- **中期（31-100场）**: 59.8%胜率，快速提升
- **后期（101+场）**: 61.2%胜率，稳定期

**趋势**: 持续improving，从探索期到稳定期提升9.1个百分点

## 位置专精度
```

## 7️⃣ RoleSpecializationAgent - 位置专精分析 ✨ NEW

**定位**: 单位置的专精程度评估

### 输入数据

### 核心功能

4. **Meta适应和gap识别**：缺少哪些meta英雄
5. **英雄池扩展建议**：推荐补充的英雄

### 输出示例

```markdown
## 掌握度评分: A

- 坦克型中单（无高坦度选择）
```

### 独特价值
- ✅ 位置维度分析（而非时间或英雄维度）
- ✅ 英雄池完整评估（广度+深度+多样性）
- ✅ Meta gap识别（缺失的英雄类型）
- ✅ 位置级别评分（专精程度量化）

## 9️⃣ WeaknessAnalysisAgent - 弱点诊断分析 ✨ NEW

## 🔟 PeerComparisonAgent - 同段位对比分析 ✨ NEW

**定位**: 相对定位和差距识别

- 玩家数据 + 同段位基准数据
- **注意**: 需要Gold layer的段位基准数据（当前使用模拟数据）

### 核心功能
1. **整体对比**：胜率、KDA vs 同段位平均水平
2. **优势领域识别**：显著强于平均的方面（Z-score > 0.5）
3. **劣势领域识别**：显著弱于平均的方面（Z-score < -0.5）
4. **段位匹配度评估**：基于表现预测合理段位
5. **Z-score标准化对比**：消除绝对值差异，聚焦相对表现

### 输出示例
```markdown
# 同段位对比报告 - GOLD

## 整体评估: Above Average

**玩家胜率**: 54.2%
**段位平均**: 50.0%
**差值**: +4.2%

## 优势领域
- **英雄池深度**: Z-score +0.8（显著高于平均）
- **版本适应速度**: Z-score +0.6（高于平均）

## 劣势领域
- **对线CS**: Z-score -0.4（略低于平均）
- **视野控制**: Z-score -0.7（显著低于平均）
```

### 独特价值
- ✅ 相对评价（vs绝对评价）
- ✅ 段位校准（合理定位）
- ✅ 优劣势清晰（聚焦差距）
- ⚠️ 需要额外数据（Gold layer段位基准）

**结论**: 11个agents构成完整的玩家分析体系，覆盖时间维度（单场→年度）和分析维度（英雄/位置/诊断/对比/推荐），没有功能重复。

### 原始5个Agents - 时间维度完整
- **PostgameReviewAgent**: 单场比赛诊断
- **DetailedAnalysisAgent**: 单版本深度剖析
- **VersionComparisonAgent**: 双版本横向对比
- **MultiVersionAgent**: 3-9版本趋势分析
- **AnnualSummaryAgent**: 40-50版本年度总结

### 新增6个Agents - 维度扩展
- **ChampionMasteryAgent**: 英雄维度（单英雄掌握度）
- **RoleSpecializationAgent**: 位置维度（单位置专精度）
- **ProgressTrackerAgent**: 动态维度（近期进步速度）
- **WeaknessAnalysisAgent**: 诊断维度（弱点识别）
- **PeerComparisonAgent**: 对比维度（vs同段位）
- **ChampionRecommendationAgent**: 推荐维度（新英雄建议）
